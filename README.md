# Minería de Datos - Repositorio de Talleres

Este repositorio está destinado a que los estudiantes del curso de **Minería de Datos** almacenen y organicen sus talleres a lo largo del semestre.

## Objetivo

Centralizar la entrega y el seguimiento de los talleres del curso en un único repositorio, manteniendo una estructura ordenada y fácil de revisar.

## Estructura sugerida

Cada estudiante debe crear una carpeta con su nombre o código y, dentro de ella, una subcarpeta para cada taller.

Ejemplo:

```text
.
├── nombre_apellido_o_codigo/
│   ├── taller_01/
│   ├── taller_02/
│   └── taller_03/
```

## Recomendaciones para la entrega

- Crear una carpeta principal identificable con el nombre del estudiante o su código.
- Nombrar los talleres de forma consistente, por ejemplo: `taller_01`, `taller_02`, etc.
- Incluir dentro de cada taller todos los archivos necesarios para su revisión.
- Agregar un archivo `README.md` dentro de cada taller si se requiere explicar el contenido, la ejecución o las decisiones tomadas.
- Evitar subir archivos innecesarios o demasiado pesados.

## Contenido esperado por taller

Dependiendo de las instrucciones dadas en clase, cada carpeta de taller puede incluir:

- notebooks en Jupyter (`.ipynb`)
- scripts en Python (`.py`)
- bases de datos pequeñas o archivos de apoyo
- gráficas, reportes o resultados
- un `README.md` con descripción breve del trabajo

## Buenas prácticas

- Mantener una estructura limpia y ordenada.
- Usar nombres de archivo claros y descriptivos.
- Verificar que los notebooks y scripts ejecuten correctamente antes de subirlos.
- Incluir únicamente el material correspondiente al taller.

## Nota

La organización final del repositorio puede ajustarse según las indicaciones del docente durante el curso.
